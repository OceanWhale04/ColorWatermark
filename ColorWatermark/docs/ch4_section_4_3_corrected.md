## 4.3.1 水印嵌入流程

水印嵌入流程的输入为彩色载体图像 **I** 和二值水印图像 **W**（工程中缩放至 **32×32** 并二值化），输出为含水印彩色图像 **Iw**。嵌入强度参数 **α** 控制水印能量注入幅度，从而影响透明性与鲁棒性之间的权衡。

嵌入环节由 **`WatermarkCodec::embed`** 完成：**不进行** CLAHE、维纳等退化预处理（预处理仅在提取模式下用于域对齐）。对每个颜色通道独立执行相同的嵌入流水线；同一水印比特序列在 **R、G、B** 三通道各嵌入一遍，以便提取阶段利用多通道信息进行融合判决。

**公式 4-1（最大奇异值加性嵌入，双极性映射）**  

实现中将二值水印像素按行优先展平为长度为 **1024** 的序列，并将每个比特 **w_bin ∈ {0, 1}** 映射为双极性符号  

**b = 2·w_bin − 1 ∈ {−1, +1}**。  

对每个 **8×8 DCT 块** 的第一个奇异值 **σ**（即最大奇异值），嵌入规则为  

**σ′ = σ + α · b。**　　　　　　　　　　　　　　　　（公式 4-1）

若正文仅用二值变量表述，应明确 **与上述双极性形式的等价写法**，例如 **σ′ = σ + α·(2w−1)**。

---

**算法 4-1 水印嵌入（DWT-DCT-SVD）**

**输入：** 彩色载体图像 **I**，二值水印图像 **W**，嵌入强度 **α**。  

**输出：** 含水印彩色图像 **Iw**。

**Begin**  

1. 将 **W** 缩放至 **32×32**，二值化；按行优先展平为长度 **1024** 的序列，映射为双极性序列 **{b_k}**，**b_k ∈ {−1, +1}**，**k = 1,…,1024**。  

2. 将 **I** 分离为 **R、G、B** 三个通道；各通道像素归一化到 **[0, 1]** 浮点表示。  

3. **对每个通道（以下步骤在三通道上重复执行）：**  

   **(a)** 对该通道执行 **一层 Haar DWT**，取低频近似子带 **LL**；  

   **(b)** 将 **LL** 划分为 **8×8** 互不重叠子块；对每个子块做 **DCT**，再做 **SVD**，提取 **最大奇异值**，按块位置排列成与分块网格对齐的标量矩阵 **Σ**（尺寸取决于 **LL** 的行列块数）；  

   **(c)** 按块遍历顺序将 **Σ** 中各系数与 **{b_k}** 按序号对应，按 **公式 4-1** 得到修改后的 **σ′**；  

   **(d)** 在每个块内用 **σ′** 替换原先的 **最大奇异值**，执行逆 **SVD**、逆 **DCT**，将重构块写回 **LL**；  

   **(e)** 对更新后的 **LL** 及其余高频子带执行 **一层逆 Haar DWT（IDWT）**，得到该通道含水印的浮点输出；再映射回 **[0, 255]** 并量化为 **8 bit**。  

4. 将三个通道 **merge** 为彩色图像 **Iw**，输出。  

**End**

**图 4-5 水印嵌入业务流程图**

图 4-5 给出了与 **`WatermarkCodec::embed`** 一致的嵌入顺序：通道分离 →（按通道）一层 **DWT** 取 **LL** → **LL** 上 **8×8** 分块的 **DCT**、**SVD** → 按公式 **4-1** 修改最大奇异值 → **逆 SVD / 逆 DCT / IDWT** → 三通道合并。**循环结构**表示 **R、G、B** 共用同一比特序列的嵌入，为提取端的多通道融合提供冗余。

## 4.3.2 水印提取与预处理融合流程

水印提取流程由 **`WatermarkSystem`** 根据 **`--preprocess`**（**none** / **clahe** / **wiener**）决定是否先做预处理，再调用 **`WatermarkCodec::extract`**。提取属 **非盲** 情形：需同时提供 **待检测图像 I′**（可能含退化）与 **原始载体图像 I_orig**，在相同变换域下比较奇异值差异。

为消除「仅增强一侧」导致的特征空间不一致，当 **`preprocess` 为 CLAHE 或维纳复原时，程序对 I′ 与 I_orig 施加同类、同参数预处理（域对齐）；** **`none`** 时二者均保持原图。

**公式 4-2（奇异值差分估计）**  

对任一通道、任一分块，记待检测块与原始块在 **DCT 域 SVD** 后得到的 **最大奇异值** 分别为 **σ′**、**σ**（标量）。水印嵌入满足公式 **4-1** 时，提取端按下式得到该位置的 **实数估计**：  

**ê = (σ′ − σ) / α。**　　　　　　　　　　　　　　　　（公式 4-2）

注意：**括号不可省略**，以免与 **σ′ − σ/α** 混淆。

---

**算法 4-2 水印提取（含预处理融合）**

**输入：** 待检测图像 **I′**，原始载体 **I_orig**，嵌入强度 **α**，预处理类型 **type ∈ {none, clahe, wiener}**（及维纳分支所需的 **PSF 参数、K** 等与命令行一致）。  

**输出：** 提取的二值水印图像 **W_ext**（**32×32**）。

**Begin**  

1. **预处理融合（域对齐）：**  

   **if** **type == clahe** **then**  

   **I′_proc ← CLAHE(I′)**，**I_orig_proc ← CLAHE(I_orig)**（同 **clipLimit、tileSize**）；  

   **else if** **type == wiener** **then**  

   由相同运动模糊参数生成 **PSF**，**I′_proc ← Wiener(I′, PSF, K)**，**I_orig_proc ← Wiener(I_orig, PSF, K)**；  

   **else**  

   **I′_proc ← I′**，**I_orig_proc ← I_orig**。  

2. 将 **I′_proc** 与 **I_orig_proc** 分别分离为 **R、G、B** 通道；各通道归一化到 **[0, 1]**。  

3. **对每个颜色通道：**  

   **(a)** 对该通道的 **I′_proc** 与 **I_orig_proc** 各自独立执行 **一层 DWT**，得到各自的 **LL**；  

   **(b)** 对两侧 **LL** 进行相同的 **8×8** 分块 **DCT** 与 **SVD**，得到逐块 **最大奇异值**，分别记为 **σ′**（来自待检测）与 **σ**（来自原始）；  

   **(c)** 对每个对应块按 **公式 4-2** 计算 **ê**，并将 **ê** 累加到全局长度为 **1024** 的累加器 **Vote** 的对应下标（三通道对同一比特位置分别贡献一项）。  

4. 令 **Vote ← Vote / 3**（对三通道估计做 **算术平均**，而非多数表决）。对每个比特位置 **k**：若 **Vote(k) ≥ 0**，则输出二值 **255**，否则 **0**（与嵌入端 **{−1,+1}** 对应的 **零阈值判决**）。  

5. 将长度为 **1024** 的序列 **reshape** 为 **32×32**，得到 **W_ext**，输出。  

**End**

**图 4-6 水印提取与预处理融合流程图**

图 4-6 对应 **`WatermarkSystem::run(extract)`** 与 **`WatermarkCodec::extract`**：**阶段一**为 **`preprocess`** 分支下的 **双侧** CLAHE 或维纳（或直通）；**阶段二**为双路径 **DWT → LL → 8×8 DCT-SVD**，在 **σ′** 与 **σ** 上按 **公式 4-2** 差分；**阶段三**为三通道累加平均与 **≥ 0** 阈值二值化及 **reshape**。与图 4-5 相比，提取流程在结构上对称，但增加了 **预处理开关** 与 **双图像奇异值对齐**。
